## 📝 작성 원칙 (참조 PRD에서 파악한 패턴)

### 1️⃣ **구체성**
- ❌ "사용자 친화적인 UI"
- ✅ "모든 버튼은 최소 44x44px, 터치 인터페이스 고려"

### 2️⃣ **측정 가능성**
- ❌ "많은 사용자 확보"
- ✅ "첫 3개월 내 MAU 500명 달성"

### 3️⃣ **사용자 중심**
- 페르소나 기반으로 기능 설명
- "김영수는 퇴근 후 30분만 학습할 수 있으므로..."

### 4️⃣ **기술적 세부사항**
- 단순 "데이터 저장" 아니라 "Firestore posts 컬렉션에..."
- 구현 방법까지 명시

### 5️⃣ **우선순위 명확화**
- Must Have / Should Have / Could Have 구분
- 이유와 함께 설명

### 6️⃣ **제약사항 명시**
- "Vercel 무료 플랜 100GB 대역폭 제한"
- 제약 내 해결 방법 제시

## 🔍 SajuFortune 프로젝트 적용 계획

### Phase 1: 섹션 1-3 (프로젝트 기초)
```
✅ 1. 프로젝트 개요 작성
   - 사주풀이 서비스의 배경과 문제
   - 무료 서비스의 목표
   - 성공 지표 정의

✅ 2. 타겟 사용자 작성
   - 페르소나 3-5개 (연령대별)
   - 사용 시나리오

✅ 3. 핵심 기능 명세
   - Must: 사주 계산, 결과 표시, PDF
   - Should: 궁합, 대운, 격국
   - Could: AI 해석, 커뮤니티
```

### Phase 2: 섹션 4-6 (기술 설계)
```
✅ 4. 사용자 플로우
   - 사주 입력 → 계산 → 결과 확인
   - 후원 플로우

✅ 5. 정보 구조/화면 설계
   - 현재 구현된 페이지 기반
   - 컴포넌트 상세 명세

✅ 6. 기술 스택/아키텍처
   - 현재 스택 문서화
   - 시스템 아키텍처 다이어그램
```

### Phase 3: 섹션 7-9 (운영 계획)
```
✅ 7. 개발 일정/마일스톤
   - 이미 완료된 작업 정리
   - 향후 로드맵

✅ 8. 위험 요소/완화
   - 기술적 리스크
   - 비즈니스 리스크

✅ 9. 성공 기준/KPI
   - 사용자 수, 체류 시간
   - 후원 전환율
```

### Phase 4: 섹션 10-15 (상세 명세)
```
✅ 10-15. 부록/API/테스트/배포
   - API 엔드포인트 문서화
```
